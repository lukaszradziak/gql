GraphQL

Nowoczesne API dla Twoich aplikacji

Prezentacja: ~20 slajdów / ~30 minut

Czym jest GraphQL?

  • Język zapytań do API stworzony przez Facebook
  • Alternatywa lub uzupełnienie dla REST
  • Umożliwia klientom precyzyjnie pobierać dane, jakich potrzebują

Główne zalety

  • Mniej "overfetchingu" i "underfetchingu"
  • Jeden endpoint dla wielu rodzajów zapytań
  • Silna typizacja schematu
  • Uproszczenie komunikacji klient-serwer

Podstawowa składnia zapytań


query {
  user(id: "1") {
    id
    name
    email
  }
}

Tylko te pola, które chcesz otrzymać.

Mutacje

Służą do modyfikacji danych, np. tworzenia, edycji, usuwania.


mutation {
  createUser(name: "Alice", email: "alice@example.com") {
    id
    name
  }
}

Subskrypcje

  • Pozwalają na komunikację w czasie rzeczywistym (np. WebSocket)
  • Doskonałe do aplikacji typu chat, live updates
  • Struktura zapytań podobna do query, ale używamy słowa kluczowego subscription

Schemat w GraphQL

Serce każdego serwera GraphQL to schemat definiujący typy i zależności:


type User {
  id: ID!
  name: String!
  email: String!
}

type Query {
  user(id: ID!): User
}

Typy i skalary

  • Podstawowe skalary: Int, Float, String, Boolean, ID
  • Własne typy obiektów (Custom Types)
  • Możliwość definiowania interfejsów i unii

Zmienne w zapytaniach


query GetUser($userId: ID!) {
  user(id: $userId) {
    id
    name
  }
}

Pozwalają dynamicznie przekazywać argumenty przy wywołaniu query.

Porównanie z REST

  • Brak konieczności tworzenia wielu endpointów
  • Precyzyjne pobieranie danych (redukcja liczby zapytań)
  • Może być trudniejsze w konfiguracji początkowej

Apollo Server (Node.js)

  • Popularna implementacja serwera GraphQL w ekosystemie JavaScript
  • Integracja z Express, Koa i innymi frameworkami
  • Łatwe definiowanie schematów i resolverów

GraphiQL / GraphQL Playground

  • Interaktywne środowiska do testowania zapytań GraphQL
  • Podpowiedzi składni i autouzupełnianie
  • Bardzo przydatne w fazie rozwoju API

Implementacje w różnych językach

  • JavaScript/TypeScript – Apollo, GraphQL.js
  • Python – Graphene
  • Ruby – graphql-ruby
  • Java – GraphQL Java
  • Go – gqlgen

Najlepsze praktyki

  • Dobrze przemyślany schemat (przejrzysty, zrozumiały)
  • Paginacja (Relay, offset-based, cursor-based)
  • Caching i batching zapytań (dataloader)
  • Mechanizmy autoryzacji i kontroli dostępu

Wydajność i bezpieczeństwo

  • Limity głębokości zapytań (depth limiting)
  • Persisted Queries (unikanie zbyt długich zapytań w URL)
  • Kontrola autoryzacji na poziomie resolverów

Federation & Gateway

  • Łączenie wielu mikroserwisów w jeden federowany schemat
  • Apollo Federation jako popularne rozwiązanie
  • Ułatwia rozwijanie większych projektów

GraphQL na produkcji

  • Monitoring zapytań (np. Apollo Studio, GraphQL Hive)
  • Optymalizacja resolverów
  • Stosowanie CDN i cachingu tam, gdzie to możliwe

Narzędzia klienckie

  • Apollo Client (React, Vue, Angular)
  • Relay (Facebook)
  • Urql (lekka alternatywa dla Apollo Client)
  • Inne biblioteki do różnych frameworków

Przykładowe scenariusze użycia

  • Aplikacje mobilne i webowe (jeden endpoint, różne potrzeby danych)
  • Microservices – GraphQL Gateway
  • Integracja z IoT i systemami real-time
